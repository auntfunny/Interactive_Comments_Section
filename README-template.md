## Overview

### The challenge

Users should be able to:

- View the optimal layout for the app depending on their device's screen size
- See hover states for all interactive elements on the page
- Create, Read, Update, and Delete comments and replies
- Upvote and downvote comments
- **Bonus**: If you're building a purely front-end project, use `localStorage` to save the current state in the browser that persists when the browser is refreshed.
- **Bonus**: Instead of using the `createdAt` strings from the `data.json` file, try using timestamps and dynamically track the time since the comment or reply was posted.

### What I learned

This was my second projected created with React. I learned how to use states and effects, and the benefits of React and its virtual DOM. I learned how to created functions and conditional rendering within JSX commponents, and overal improved my understand of the React library. I also learned more of the utility of the React component, and always being sure that it can be reused.

### Continued development

I still need to continue to improve my comfort with React, and my knowledge of the different hooks that it provides. Every time the delete button is pressed, the entire list re-renders, and I could not figure out how to get around that issue. I want to learn about the useContext hook to avoid the prop drilling that I needed to use for this project. I also want to learn better ways to re-render components when a data source is changed, instead of forcing the re-render.
